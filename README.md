# 💰 บัญชีเงินเบียร์ - Web Application บันทึกรายการฝาก–ถอนเงิน

ระบบเว็บแอปพลิเคชัน (Web Application) สำหรับบริหารจัดการรายการฝาก-ถอนเงินที่ออกแบบในสไตล์ **SaaS Dashboard (Glassmorphic Design)** โดยใช้ **Google Sheet เป็นฐานข้อมูลหลัก** และมีระบบส่งอีเมลแจ้งเตือนทุกครั้งที่มีการเพิ่มรายการธุรกรรมใหม่

---

## ✨ คุณสมบัติของระบบ

- **หน้าสรุปข้อมูลการเงิน (Dashboard)**: แสดงยอดเงินคงเหลือปัจจุบัน ยอดฝากสะสม ยอดถอนสะสม และจำนวนรายการทั้งหมด
- **ยอดเงินตั้งต้นอัตโนมัติ**: คำนวณยอดเงินสะสมอย่างถูกต้องโดยอิงจากเงินคงเหลือเริ่มต้น **44,540.83 บาท**
- **กราฟสรุปสถิติ**:
  - กราฟแท่ง (Bar Chart) สรุปยอดฝากและยอดถอนเปรียบเทียบรายเดือน
  - กราฟเส้น (Line Chart) แสดงแนวโน้มยอดเงินคงเหลือสะสมตามลำดับเวลา
- **ระบบ CRUD**: รองรับการ เพิ่ม (Add), แก้ไข (Update) และ ลบ (Delete) ข้อมูลการทำธุรกรรม โดยจะซิงค์ข้อมูลลง Google Sheet ทันที
- **ระบบป้องกันข้อมูลซ้ำ (Duplicate Prevention)**: ป้องกันการกดบันทึกซ้ำจากฝั่งผู้ใช้ (Double Click) และตรวจสอบค่า Timestamp ที่ซ้ำกัน
- **ระบบอีเมลแจ้งเตือนอัตโนมัติ**: ส่งอีเมลแจ้งเตือนในรูปแบบ Template สวยงาม ไปยัง:
  - `apinya04042528@gmail.com`
  - `puisena88@gmail.com`

## 🛠️ โครงสร้างไฟล์ในโครงการ

1. **`index.html`** - หน้าโครงสร้าง UI ของระบบ คอนเทนเนอร์ และฟอร์มป๊อปอัป
2. **`style.css`** - สไตล์การออกแบบ Glassmorphism การจัดเลย์เอาต์ที่รองรับ Responsive และแอนิเมชัน
3. **`app.js`** - ส่วนควบคุมการทำงานฝั่งไคลเอนต์ การดึงข้อมูล วาดกราฟ ค้นหาตัวกรอง และส่ง API
4. **`Code.gs`** - โค้ดหลังบ้านของ Google Apps Script สำหรับใส่ในตัวแก้ไข Apps Script ของ Google Sheet

## 🚀 ขั้นตอนการติดตั้งและใช้งานระบบ

ทำตามขั้นตอนด้านล่างนี้เพื่อเชื่อมต่อเว็บแอปพลิเคชันเข้ากับ Google Sheet ของคุณ:

### ขั้นตอนที่ 1: เตรียม Google Sheet
1. เข้าไปที่ [Google Sheets](https://sheets.google.com) และสร้างตารางงานใหม่ (Spreadsheet เปล่า)
2. คุณสามารถตั้งชื่อไฟล์ Sheet เป็นอะไรก็ได้ เช่น **"บัญชีเงินเบียร์"**

### ขั้นตอนที่ 2: ตั้งค่า Google Apps Script
1. ที่แถบเมนูด้านบนของ Google Sheet ให้เลือก **ส่วนขยาย (Extensions)** &rarr; **Apps Script**
2. ลบโค้ดเริ่มต้นในไฟล์ `Code.gs` ออกทั้งหมด
3. เปิดไฟล์ **[Code.gs](Code.gs)** ในโฟลเดอร์โครงการนี้ และคัดลอกโค้ดทั้งหมดไปวางลงในหน้าต่าง Apps Script
4. กดปุ่มบันทึกโครงการ (รูปแผ่นดิสก์) หรือกดปุ่ม `Ctrl + S`

### ขั้นตอนที่ 3: ทำให้ใช้งานได้ในฐานะเว็บแอป (Deployment)
1. กดปุ่ม **ทำให้ใช้งานได้ (Deploy)** ที่มุมขวาบนของหน้าต่าง Apps Script &rarr; เลือก **การปรับใช้ใหม่ (New deployment)**
2. ในป๊อปอัป ให้คลิกรูปฟันเฟืองข้าง "เลือกประเภท (Select type)" และเลือก **เว็บแอป (Web App)**
3. ตั้งค่าการกำหนดค่าดังนี้:
   - **คำอธิบาย (Description)**: *เขียนคำอธิบายตามต้องการ เช่น "บัญชีเงินเบียร์ v1"*
   - **เรียกใช้ในฐานะ (Execute as)**: เลือก **ฉัน (อีเมลของคุณ - your-email@gmail.com)**
   - **ผู้มีสิทธิ์เข้าถึง (Who has access)**: เลือก **ทุกคน (Anyone)** *(สำคัญมาก! เพื่อให้หน้าเว็บสามารถเรียกใช้ API ได้)*
4. กดปุ่ม **ทำให้ใช้งานได้ (Deploy)**
5. ระบบจะแสดงหน้าต่างให้สิทธิ์การเข้าถึง (Authorize Access)
   - คลิกปุ่ม **ให้สิทธิ์เข้าถึง (Authorize Access)**
   - เลือกบัญชี Google ของคุณ
   - หากเจอหน้าแจ้งเตือนความปลอดภัย (Google hasn't verified this app) ให้คลิกที่คำว่า **ขั้นสูง (Advanced)** &rarr; คลิกลิงก์ **ไปที่... (ไม่ปลอดภัย) / Go to... (unsafe)**
   - กดปุ่ม **อนุญาต (Allow)**
6. เมื่อการติดตั้งเสร็จสมบูรณ์ คุณไม่จำเป็นต้องคัดลอก URL นี้ไปใส่ในระบบ เนื่องจากผู้พัฒนาได้ทำการตั้งค่าเชื่อมโยง URL ตัวนี้เข้าไปในระบบให้คุณแบบอัตโนมัติแล้ว!

### ขั้นตอนที่ 4: เปิดใช้งานหน้าเว็บและทดสอบ
1. เปิดไฟล์ `index.html` บนบราวเซอร์ของคุณ (สามารถดับเบิ้ลคลิกเพื่อเปิด หรือเปิดผ่าน Local Live Server)
2. ระบบจะโหลดข้อมูลแดชบอร์ด ยอดเงินคงเหลือเริ่มต้น 44,540.83 บาท และรายการต่างๆ โดยอัตโนมัติ
3. เมื่อคลิกที่ปุ่ม **"บันทึกรายการ"** หรือ **"บันทึกรายการใหม่"** ป๊อปอัปฟอร์มบันทึกข้อมูลจะแสดงขึ้นมา
4. **ระบบความปลอดภัย**: ในการบันทึกรายการ (ทั้งการเพิ่มข้อมูลและการแก้ไขข้อมูล) รวมถึงการลบข้อมูล ระบบจะให้กรอกรหัสเข้าใช้งานดังนี้:
   - **รหัสผ่าน: `Admin1234`**
5. เมื่อยืนยันรหัสถูกต้อง รายการจะถูกส่งไปบันทึกบน Google Sheet และระบบจะส่งอีเมลแจ้งเตือนอัตโนมัติ พร้อมอัปเดตสถิติยอดคงเหลือสะสมบนแดชบอร์ดทันที

## 💡 ข้อมูลเพิ่มเติมเกี่ยวกับการป้องกันข้อมูลซ้ำ (Duplicate Prevention)

ระบบได้ออกแบบขั้นตอนป้องกันการบันทึกข้อมูลธุรกรรมซ้ำซ้อน 2 ระดับ:
1. **ระดับไคลเอนต์ (Client-Side)**: ปุ่มส่งฟอร์ม (บันทึกรายการ) จะถูกปิดการใช้งาน (disabled) ทันทีหลังกดส่ง และเปลี่ยนสถานะเป็น Loading เพื่อป้องกันไม่ให้ผู้ใช้งานกดซ้ำรัวๆ
2. **ระดับเซิร์ฟเวอร์ (Server-Side/Apps Script)**: ก่อนที่จะทำการเพิ่มแถวข้อมูลลงใน Google Sheet โค้ด Apps Script จะทำการตรวจสอบ:
   - ตรวจสอบ `Timestamp` (ซึ่งทำงานเสมือน Primary Key เฉพาะของธุรกรรมนั้นๆ)
   - ตรวจสอบหากมีแถวข้อมูลล่าสุดที่มีข้อมูลเหมือนกัน (วันที่ ประเภท จำนวนเงิน รายละเอียด ผู้บันทึก) และถูกบันทึกเข้ามาภายในระยะเวลาไม่เกิน 10 วินาที ระบบจะปฏิเสธการบันทึกพร้อมตอบกลับสถานะ `duplicate` ไปยังหน้าเว็บ
